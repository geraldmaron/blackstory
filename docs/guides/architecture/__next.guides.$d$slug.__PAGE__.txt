2:Te34,<h1 id="architecture"><a class="heading-anchor" href="#architecture">Architecture</a></h1>
<p>BlackStory is a place-connected Black history research platform. Public surfaces
serve only released historical projections. Research, evidence, and promotion
stay behind private APIs, workers, and admin tools.</p>
<h2 id="surfaces"><a class="heading-anchor" href="#surfaces">Surfaces</a></h2>
<pre><code>apps/web                 Public Next.js (Vercel; see ADR-027)
apps/admin               Private admin / research console
apps/api-public          Public read / search / location
apps/api-submissions     Corrections / contribution intake
apps/api-internal        Publication / promotion (private)
apps/mobile              Expo mobile app (isolated lockfile)
apps/docs                This GitHub Pages site
workers/*                Research, publication, security
functions/               Tombstone (discovery schedules: ADR-028)
packages/*               Shared libraries (@repo/*)
supabase/                Postgres migrations and project config
infra/*                  Firebase wind-down, GCP, GitHub scaffolding
</code></pre>
<p>Do not add deployable microservices beyond this set. See ADR-005 in the
repository.</p>
<h2 id="data-plane"><a class="heading-anchor" href="#data-plane">Data plane</a></h2>
<p><strong>Product system of record</strong> is Supabase Postgres (<code>blackstory-app</code>), with
schema and migrations under <code>supabase/migrations/</code> and
<code>docs/data/postgres-schema.md</code>. Decision record: ADR-020.</p>
<p>Public and admin apps read with <code>PUBLIC_DATA_SOURCE=postgres</code> /
<code>ADMIN_DATA_SOURCE=postgres</code>. Published developer reads may use PostgREST views
(ADR-026). Blobs: Supabase Storage for <code>public-media</code> with GCS dual-serve during
wind-down. Firestore is export/rollback only
(<code>docs/data/firebase-wind-down.md</code>).</p>
<p>Local PostGIS under <code>infra/database/</code> is parked scaffolding, not the production
path. Cloud SQL and SQL Connect are permanently non-path (ADR-002 / ADR-003
superseded).</p>

<h2 id="product-constitution"><a class="heading-anchor" href="#product-constitution">Product constitution</a></h2>
<p>Shared policy lives in <code>packages/schemas/constitution/policy.v1.json</code>. TypeScript
loads it through <code>@repo/schemas</code>. Python workers load the same JSON through
<code>black_book_constitution</code>. Thresholds and living-person rules change by version
bump, not by an HTTP write API.</p>
