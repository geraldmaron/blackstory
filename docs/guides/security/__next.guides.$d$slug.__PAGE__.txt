2:T8de,<h1 id="security-posture"><a class="heading-anchor" href="#security-posture">Security posture</a></h1>
<p>BlackStory assumes a hostile environment for public surfaces and contribution
intake. The product promise depends on that assumption: a public page is only as
trustworthy as the gates between research and release.</p>
<h2 id="defaults"><a class="heading-anchor" href="#defaults">Defaults</a></h2>
<ul>
<li>Public clients never write canonical history.</li>
<li>Submissions land in quarantine; compromise of intake must not equal publish.</li>
<li>Research credentials cannot promote or deploy.</li>
<li>Living residential addresses are never returned publicly; unknown living
status is treated as living.</li>
<li>Production identifiers fail closed in local tests (<code>pnpm test:preflight</code>).</li>
<li>GitHub Actions stay read-only by default; third-party actions pin commit SHAs.</li>
<li>External URLs are untrusted; public request paths do not synchronously fetch
arbitrary remote content.</li>
</ul>
