# Privacy incident

## Trigger and triage

- Trigger on protected/living-person data in public output, logs, exports, caches, uploads, or unauthorized access.
- Notify privacy/legal leadership immediately; minimize access and record jurisdictions, subjects, fields, and exposure window.

## Contain

1. Engage `publication`, geocoding, nearby, exports, search, and uploads.
2. Enter static mode only if the active immutable snapshot is safe; otherwise perform immediate BB-019 rollback first.
3. Revoke affected access independently and quarantine exposed records/artifacts.
4. Purge affected CDN/cache/log copies where feasible without destroying required evidence.

## Recover

- Correct through audited retraction/new release, verify public DTO absence, and inspect all derivative stores.
- Follow legal notification timelines; never include unnecessary personal data in notices or tickets.
- Add a sanitized regression fixture and require privacy approval before re-enable.
