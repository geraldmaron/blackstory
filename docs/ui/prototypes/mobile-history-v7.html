Mobile History v7 — polish constraints

Binding: design-direction-v6-mobile.md, design-direction-v6-history.md, Explore v7 lessons in mobile-explore-v7.html.

  • 16dp horizontal edge inset (screenScrollInsets.paddingHorizontal / space['4']).
  • Masthead at subtitle scale (20px Sora) in dense browse mode — not display (34px).
  • Category browse uses full-width ledger rows — never a two-up grid that mid-word wraps long kind labels.
  • Continuous Surface: sectioning via SectionHeader + hairline dividers — not stacked bordered cards inside cards.
  • Dev gear / options control lives in the masthead trailing slot (DevMenuHeaderButton), not floating over the intro panel.
  • ApiStatusBanner stays useful in __DEV__ but uses compact padding and smaller type.
  • 44dp touch targets preserved via row minHeight and ghost button size.
