Design principles applied

  • Map dominates first glance: thin mast (count + ghost controls) over edge-to-edge plate, not a branded Surface slab.
  • 16px horizontal edge inset on mast, instruments inner content, sheet/rail rows, and preview body (safe-area + gutter; RN uses screenScrollInsets / space['4'] = 16dp).
  • Continuous Surface fields for instruments and sheet; sectioning via type, spacing, and rules — not stacked bordered cards.
  • Map stays 60%+ of viewport; sheets preserve a map sliver (Google Maps 2025, Apple Maps detents).
  • Pins at national zoom use ~0.85× radius so they do not blot counties; locality zoom allows up to 11px.
  • 44px touch targets on chrome via hit padding; pin hit area can exceed visual radius.
  • Opaque Surface panels follow device theme; map plate stays fixed dark archive.
  • Copper only on active tab underline, selection rule, count accent, and one preview CTA (~10–15%).
  • Flat matte only: no shadows, glows, gradients, or blur on browse surfaces.

RN implementation target

  • Port web markerRadiusExpression() zoom scaling (done in this session).
  • Exclusive bottom sheet states: instruments OR records expanded OR preview half.
  • Extend FilterState with tone, theme, status, confidence, state (web parity).
